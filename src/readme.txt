服务框架:(事件循化中对于事件列表都为间线程操作，可以不加锁)
master线程:(暂时没有管理端口)
1.解析配置文件，得到需要监听的端口
2.启动net线程,启动work线程
3.把需要监听的端口添加为watcher,把net线程的上报队列设置为watcher,添加时间事件,每X秒打印一次统计信息
4.处理时间事件
5.收到监听端口消息调用netthread::addWatcher(),把新链接通过队列传给netthread
6.收到上报消息,累加统计,调整负载

net线程:
1.把master的下行队列添加为watcher,把worker的上行队列添加为watcher,添加时间事件毎X秒上报一次统计信息(链接数,处理请求数,各接口平均处理时间,错误数)
2.处理时间事件,如果是请求则给客户端返回超时
3.收到master的下行请求
    3.1 新链接,把新链接添加为watcher
4.收到客户端发来的请求,recv数据,recv完整包后调用workthread::addEvent,把请求通过队列传给workerthread,并且如果请求需要返回把recv到请求的加入时间队列
5.收到worker的上行请求
    1.客户端请求返回数据,如果时间事件中已经没有(有超时设置)就丢弃
    2.请求其它服务,发送请求,如果还没有连接就连接

work线程:
1.把net的下行队列添加为watcher
2.收到net的下行请求
    2.1 处理请求
3.调用其它服务
    3.1 同步调用,通过上行队列把请求发给netthread,结构中带返回信息
    3.2
